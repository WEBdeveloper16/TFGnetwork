Arquitectura de Red en Árbol

Estructura jerárquica de tres capas para máximo rendimiento

Capa Núcleo (Core)

Switches de alto rendimiento que interconectan los switches de distribución

  • Switch multilayer 3560
  • Alta velocidad
  • Redundancia

Capa Distribución

Switches que agregan y segmentan el tráfico de los switches de acceso

  • Switches 2960
  • Enrutamiento Inter-VLAN
  • Políticas de seguridad

Capa Acceso

Switches que conectan directamente a los dispositivos finales

  • Switches 2960 con PoE
  • Teléfonos IP
  • Puntos de acceso Wi-Fi

Configuración Paso a Paso

Instrucciones detalladas para implementar la red empresarial

Dispositivos necesarios:
  • Switch Núcleo: 3560 Multilayer
  • Switch Distribución: 2960 (2 unidades)
  • Switch Acceso: 2960 con PoE (8 unidades)
  • Router: 2811 para Internet
  • Access Points: AP-PT (15 unidades)
Tip Importante

Utiliza cables de fibra óptica entre switches de núcleo y distribución para mayor velocidad.

Asignar puertos a VLANs:
interface fastEthernet 0/1
 switchport mode access
 switchport access vlan 10
exit

interface fastEthernet 0/2
 switchport mode access
 switchport access vlan 20
exit

Configurar enlaces troncales:
interface GigabitEthernet0/1
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan all
 switchport trunk native vlan 1
exit

interface GigabitEthernet0/2
 switchport trunk encapsulation dot1q
 switchport mode trunk
 switchport trunk allowed vlan all
 switchport trunk native vlan 1
exit
Importante: Configura troncales en todos los enlaces entre switches para permitir el paso de múltiples VLANs.

Configuración de Telefonía IP

Implementación completa del sistema de comunicaciones VoIP

Call Manager Express

Servidor de telefonía integrado en router Cisco para gestión de llamadas

Teléfonos IP

Dispositivos IP Phone conectados via PoE para cada departamento

Infraestructura PoE

Power over Ethernet para alimentar teléfonos sin cables adicionales

Configuración Detallada

1. Configuración del Call Manager Express en Router:
configure terminal

! Habilitar telefonía IP
telephony-service
 max-ephones 50
 max-dn 50
 ip source-address 192.168.1.1 port 2000
 auto-reg-ephone
 create cnf-files version-stamp 7960 Jun 30 2024 00:00:00

! Configurar DHCP para teléfonos IP
ip dhcp pool VOICE_POOL
 network 192.168.1.0 255.255.255.0
 default-router 192.168.1.1
 option 150 ip 192.168.1.1
 dns-server 8.8.8.8

! Configurar extensiones
ephone-dn 1
 number 1001
 name "Direccion General"

ephone-dn 2
 number 1002
 name "Recursos Humanos"

ephone-dn 3
 number 1003
 name "Finanzas"

ephone-dn 4
 number 1004
 name "Produccion"
Información Importante
  • Option 150: Dirección del servidor TFTP
  • Max-ephones: Número máximo de teléfonos
  • Auto-reg-ephone: Registro automático

Pasos para Configurar en Packet Tracer:
  1. Añadir Router 2811: Arrastrar desde panel de dispositivos
  2. Conectar al Core Switch: Cable directo GigabitEthernet
  3. Añadir Teléfonos IP: IP Phone-PT desde End Devices
  4. Conectar a Switch PoE: Cable directo al switch de acceso
  5. Configurar Router: CLI con comandos mostrados arriba
  6. Verificar Registro: show ephone y show ephone-dn

Sistema de Vigilancia IP

Implementación completa de cámaras de seguridad y monitoreo

Cámaras IP

Cámaras de alta definición con conectividad de red

NVR Server

Servidor de grabación de video en red para almacenamiento

Centro de Control

Estaciones de monitoreo para vigilancia en tiempo real

Almacenamiento

Respaldo en la nube y almacenamiento local redundante

Configuración del Sistema de Vigilancia

1. Configuración de VLAN para Vigilancia:
! Crear VLAN específica para vigilancia
vlan 200
 name VIGILANCIA_IP
exit

! Configurar interfaz VLAN para cámaras
interface vlan200
 ip address 192.168.200.1 255.255.255.0
 no shutdown
exit

! Asignar puertos a VLAN de vigilancia
interface range fastEthernet 0/20-24
 switchport mode access
 switchport access vlan 200
 power inline auto
exit
2. Configuración DHCP para Cámaras:
! Pool DHCP para cámaras IP
ip dhcp pool CAMERAS_POOL
 network 192.168.200.0 255.255.255.0
 default-router 192.168.200.1
 dns-server 192.168.200.1
 lease 7
exit

! Reservas estáticas para cámaras críticas
ip dhcp reservation
 client-identifier 01aa.bbcc.ddee.ff
 ip address 192.168.200.10
 client-name "Camara-Entrada-Principal"
exit
Consideraciones de Seguridad
  • Usar contraseñas fuertes
  • Cambiar credenciales por defecto
  • Habilitar HTTPS
  • Configurar ACLs restrictivas
  • Actualizar firmware regularmente

Acceso Web a Cámaras
Configuración de acceso web:
  1. Acceder a la IP de la cámara via navegador
  2. Usuario: admin / Pass: admin123
  3. Configurar red: IP estática, gateway, DNS
  4. Configurar video: resolución, FPS, bitrate
  5. Configurar grabación: horarios, eventos
  6. Configurar usuarios: permisos de acceso
Configuración NVR
Servidor de grabación:
  1. Instalar software NVR en servidor
  2. Configurar almacenamiento (RAID recomendado)
  3. Añadir cámaras por IP
  4. Configurar perfiles de grabación
  5. Establecer retención de datos
  6. Configurar alertas y notificaciones
Pasos en Packet Tracer:
  1. Añadir Cámaras IP: Desde IoT Devices → Generic IP Camera
  2. Conectar a Switch: Cable directo a switch PoE
  3. Configurar IP: Click en cámara → Config → Global Settings
  4. Añadir Servidor: Server-PT para NVR
  5. Configurar Software: Habilitar servicio de video
  6. Probar Conectividad: Ping desde PC de seguridad
